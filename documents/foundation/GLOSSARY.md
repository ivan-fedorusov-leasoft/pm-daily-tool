# Foundation

------------------------------------------------------------------------

## Today's Work

Primary developer workspace used to submit today's progress.

## Daily Note

A daily status update for one game.

## Daily Mode

Shared presentation mode controlled by the Team Lead.

## Radar

Compact overview of all games.

## Game Page

Detailed page representing a single game. Source of truth for project
information.

## Stage

A predefined project milestone (Playable, Alpha, Beta, etc.).

## Stage Date

Target date assigned to a stage.

## Change Request

A proposed modification submitted by a developer and approved or
rejected by a Team Lead.

## Team Lead Attention

A flag indicating that the developer requires discussion during Daily.

## Transcript

AI-generated or imported Daily meeting transcription.

## Presence

Realtime indication that another user is viewing or editing an entity.

## Lock

Temporary restriction preventing conflicting edits.

## Timeline

Chronological activity feed (future feature).

## AI Summary

Automatically generated summary produced from transcripts or project
history (future feature).

## Manager

A user with project management permissions.
Managers may also be assigned as developers to games.

## Daily Host

A Daily Host may be any user.
A temporary role assigned for one Daily session.
Controls the meeting but does not receive Manager permissions.
A Daily Host is usually the user who started the Daily session.

## Game Status

Lifecycle state of a game in the tool.
Controls whether the game appears in Today's Work and Daily Mode.
