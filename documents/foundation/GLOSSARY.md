# GLOSSARY

## Today's Work

Primary developer workspace used to submit today's progress.

## Daily Note

A daily status update for one game.

## Daily Mode

Shared presentation mode controlled by the Daily Host.

## Radar

Compact overview of all games.

## Game Page

Detailed page representing a single game.
Source of truth for project information.

## Stage

A predefined project milestone.

Examples:

- Playable
- Alpha
- Beta
- Gold
- Master

## Stage Date

Target date assigned to a stage.

## Change Request

A proposed modification submitted by a user and approved or rejected by a Manager/Admin.

## Team Lead Attention

A flag indicating that the developer requires discussion during Daily.

This name is kept as a product/UI label even though the permission role is called Manager.

## Transcript

AI-generated or imported Daily meeting transcription.

## Presence

Realtime indication that another user is viewing or editing an entity.

## Lock

Temporary restriction preventing conflicting edits.

A lock does not grant permissions.

## Timeline

Chronological activity feed.

Future feature.

## AI Summary

Automatically generated summary produced from transcripts or project history.

Future feature.

## Manager

A user with project management permissions.

Managers may also be assigned as developers to games.

## Admin

A user with full system permissions.

Admins may also be assigned as developers to games.

## Daily Host

A temporary session controller for one Daily session.

Any authenticated user may become Daily Host by starting a Daily session.

Daily Host controls the meeting but does not receive Manager/Admin permissions.

## Game Status

Lifecycle state of a game in the tool.

Controls whether the game appears in Today's Work and Daily Mode.
