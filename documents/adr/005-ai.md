# ADR-005: AI Starts in v3

## Decision

Postpone AI features until v3.

## Reason

- The core workflow must work without AI.
- AI should improve an existing process, not define the product.
- MVP should validate manual status updates, Game Page, Simple Radar, and Change Requests first.
- Transcript parsing and summaries need stable data structures.

## Alternatives

- Build AI into MVP.
- Use AI as the primary interface.
- Skip AI entirely.

## Consequences

- No transcript parsing in MVP.
- No AI summaries in MVP.
- No semantic search in MVP.
- v3 may add Gemini/ChatGPT transcript parsing, summaries, embeddings, and semantic search.
